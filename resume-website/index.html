Building Persona-Driven Demo Systems for Industrial Platforms

August 2025

When you're trying to sell a complex industrial platform—whether it's a Manufacturing Execution System (MES), a digital workflow engine, or a mobile forms app—the real magic happens when you step into your buyer's shoes. Over the past year at Adatafy, I've built demo environments for three different platforms—Shiftconnector, Augmentir, and GoCanvas—that our sales team now uses every week to win new clients. Here's how I approached each one from the perspective of key personas, the core features I highlighted, and why those demos resonate in day-to-day operations.

1. Understanding Each Platform's Strengths

Shiftconnector (MES System)

Focus: real-time shop-floor visibility, automated workflows, data capture for compliance.

Strengths:

  • Workflow module that enforces standard operating procedures (SOPs)
  • Real-time dashboards for OEE, quality, and throughput metrics
  • Integration with PLCs and sensors for automated data ingestion
Augmentir (Digital Workflows + Guided Procedures)

Focus: guided work instructions, skill-based routing, remote assistance.

Strengths:

  • Step-by-step guided procedures with embedded videos and AR annotations
  • Skill-based work assignment that routes tasks to qualified technicians
  • Built-in feedback loops to capture field-level improvement suggestions
GoCanvas (Mobile Forms & Data Collection)

Focus: flexible, code-free form building, offline data capture, custom reporting.

Strengths:

  • Drag-and-drop form builder for checklists, inspections, audits
  • Offline mode for remote or shielded industrial environments
  • Easy export & integration with BI tools (Excel, Tableau, Power BI)

2. Putting Personas First

Each demo starts by defining who we're talking to—and what pain they feel on a Monday morning:

Persona Top Pain Point Demo Focus
Operations Manager Lack of real-time visibility into line performance Shiftconnector dashboards & alerts
Maintenance Lead Unclear procedures & long repair times Augmentir's guided work instructions
Quality Engineer Paper-based inspections & audit prep headaches GoCanvas mobile forms & report exports

3. Demo Design Walk-through

A. Shiftconnector Demo

Persona: Operations Manager
Use Case: Bypass interlocks procedure at an ethanol plant

Key Features Shown:

  • Workflow Module: I configured a "Bypass Interlock" SOP that steps operators through each safety check, with digital sign-off at each stage.
  • Real-Time Dashboard: Simulated sensor data feeds into an OEE dashboard, highlighting how a single mis-step would trigger an alert.
  • Data Drill-Down: I demonstrate drilling from a high-level KPI down to the exact shift and operator who bypassed an interlock, showing traceability for compliance audits.
B. Augmentir Demo

Persona: Maintenance Lead
Use Case: Guided valve replacement on a packaging line

Key Features Shown:

  • Step-by-Step Guidance: I built a procedure with inline images and an AR overlay (simulated in the demo) that highlights the exact valve bolts to remove.
  • Skill-Routing Logic: The demo user logs in; Augmentir automatically assigns the task only if their profile has the required safety certification.
  • Feedback Capture: At the end, the user can record a quick voice note suggesting a tweak—illustrating how continuous improvement gets fed back into the product team.
C. GoCanvas Demo

Persona: Quality Engineer
Use Case: Daily line inspection & audit reporting

Key Features Shown:

  • Form Builder: I show how to drag in a photo-capture field for "Cleanliness Check," a dropdown for "Pass/Fail," and a signature block for line manager approval.
  • Offline Submission: I simulate "no network" on the tablet and show offline data being queued, then automatically uploaded when connectivity returns.
  • Custom Reports: Finally, I export the week's inspection data into a Tableau-compatible CSV, demonstrating fast audit prep.

4. Why This Approach Works

  • Persona Empathy: By framing each session around a real user's daily struggles, prospects immediately see relevance.
  • Use Case Authenticity: I research industry best practices—e.g., ethanol-plant interlock standards, packaging-line maintenance checklists—so the demo feels like "real work," not a toy.
  • Feature Prioritization: Instead of overwhelming with every toggle and report, I showcase the three features each persona cares about most.
  • Interactive & Measurable: Prospects can click through steps themselves and see how easy it is to gather data, generate reports, or escalate issues—building confidence and speeding decision-making.

Conclusion

Crafting a demo system isn't just about "turn features on." It's about telling a story that mirrors a prospect's day, speaks their language, and shows exactly how the platform will transform their workflows. Whether it's a safety-critical SOP in Shiftconnector, an AR-enhanced procedure in Augmentir, or a mobile inspection app in GoCanvas, focusing on persona pain points and common industrial use cases ensures every demo session becomes a powerful selling moment.

MES Systems 101: What Developers Should Know

August 2025

Introduction

Manufacturing Execution Systems (MES) live at the critical junction between the shop floor and IT: collecting high-frequency sensor data, enforcing Standard Operating Procedures (SOPs), and feeding real-time dashboards for plant managers. Unlike typical web apps, MES must juggle millisecond-accurate timestamps, strict audit trails, and spotty network conditions. In this post, we'll break down the core MES components in plain English—and then dive into three platforms I've worked with (Shiftconnector, AspenTech Inmation & Augmentir) to surface the hidden engineering lessons every developer should know.

1. MES Core Components Explained

A. Data Ingestion & Historian

Role: Capture real-time streams from PLCs, SCADA, and IoT sensors—often hundreds of tags per second—and store them in a time-series database (TSDB).

Why It Matters: Precise, millisecond-aligned data is non-negotiable for troubleshooting, optimization, and regulatory compliance.

B. Workflow / Procedure Engine

Role: Model SOPs as state machines. Operators can't skip steps, and every action (with user ID and timestamp) is logged.

Why It Matters: Regulated industries (pharma, energy, chemicals) require strict adherence to procedures—or face downtime, recalls, or fines.

C. Analytics & Dashboards

Role: Transform raw events into KPIs—OEE (Overall Equipment Effectiveness), MTTR (Mean Time To Repair), batch yield, quality metrics, and more.

Why It Matters: Plant managers need instant visibility into performance and quality to drive continuous improvement.

D. Integration & APIs

Role: Offer RESTful, OPC-UA, or proprietary connectors to ERP systems, historians (e.g. PI, Inmation), BI tools (Power BI, Tableau), and custom clients.

Why It Matters: MES rarely stands alone; seamless data exchange with upstream and downstream systems is essential.

2. The Hidden Engineering Challenges

High-Velocity Streams

Pitfall: Treat sensor updates like HTTP requests.

Reality: Use event-driven architectures (Kafka, RabbitMQ) or specialized TSDBs to handle thousands of writes per second.

Time Synchronization

Pitfall: Trust each device's local clock.

Reality: Implement NTP/PTP across PLCs and servers. Convert all timestamps to a single source of truth for accurate traceability.

Offline Resilience

Pitfall: Assume constant connectivity.

Reality: Buffer events at the edge (browser, gateway) and retry submissions when the network returns, ensuring no data loss.

Regulatory Audit Trails

Pitfall: Log only "success/failure."

Reality: Capture detailed metadata—operator ID, before/after values, code versions, and change reasons—for every action.

3. Platform Deep Dive

3.1 Shiftconnector: Digital Collaboration & Workflow Enforcement

What It Is: A shift-handover and collaboration tool that replaces paper logs and siloed emails with structured workflows and searchable handover records.

Core Strengths:

  • Workflow Engine: Enforces SOPs as state machines—operators can't skip or reorder steps.
  • Real-Time Collaboration: Centralized feed for shift logs, incident reporting, and formal handovers.
  • PLC/DCS Integration: Auto-trigger tasks or alerts when sensor thresholds are hit.

Developer Takeaways:

  • Pluggable Step Executors: Build a unified interface so each workflow step can call a human-confirmation handler or a PLC check with the same code path.
  • Local Event Buffering: Edge-buffer workflow events in the browser or gateway; flush when connectivity resumes.
  • Feature Toggles for Variants: Keep plant-specific SOP tweaks (e.g., "Bypass Interlock" rules) behind configuration flags instead of code forks.
3.2 AspenTech Inmation: The Industrial Data Fabric

What It Is: A scalable middleware that unifies real-time tag data, historical archives, and relational batch records into a single, secure platform.

Core Strengths:

  • Unified Connectivity: Adapters for PLC/SCADA, OSIsoft PI, relational DBs, and more.
  • Dual Storage Layers: Raw time-series data in the TSDB; aggregated KPIs in relational tables.
  • Distributed & Secure: Enterprise-grade scaling with fine-grained access controls.

Developer Takeaways:

  • Separate TSDB & OLTP: Never jam high-frequency writes into relational schemas—let the historian handle raw tags and use the relational layer for dashboards.
  • Metadata-First Modeling: Tag each data point with unit, precision, and source to avoid consistency bugs.
  • Simulator for Testing: Create a lightweight tag-value simulator to accelerate both demo setups and automated tests.
3.3 Augmentir: AI-Powered Digital Workflows

What It Is: A no-code platform that transforms paper SOPs into interactive digital instructions, enhanced with AI and AR to guide frontline technicians.

Core Strengths:

  • Drag-and-Drop Workflow Builder: Define steps, checks, media, and decision logic without coding.
  • AI Personalization: Machine-learning models recommend next best actions and identify procedural bottlenecks.
  • Remote Expert & Feedback: Technicians can request live help or submit improvement suggestions in-app.

Developer Takeaways:

  • Micro-Frontend Steps: Build each workflow step as an independent component for easy reuse and testing.
  • Instrument Everything: Log user interactions, time-to-complete, and decision paths so AI models have clean data to learn from.
  • Offline-First UX: Cache workflows on the device and sync user logs when network connectivity returns.

4. Best Practices & Wrap-Up

  • Event-Driven Architecture: Decouple ingestion from processing with message brokers to handle spikes gracefully.
  • Metadata Discipline: Always include units, precision, source, and operator IDs on your data models.
  • Pre-Production Simulators: Use synthetic data generators to build, test, and demo without a live plant.
  • Modular Workflow Design: Treat each SOP step as a standalone plugin or micro-service to simplify maintenance and site-specific variations.

MES platforms demand a specialized approach—real-time data handling, strict compliance, and unreliable networks. By applying these lessons from Shiftconnector, Inmation, and Augmentir, you'll build more resilient, auditable, and user-friendly systems that bridge engineering and operations effectively.

From Confusion to Clarity: Writing Docs That Developers Actually Use

August 2025

How clear, purpose-driven documentation accelerates team productivity

Introduction

We've all been there: a new tool lands in our stack, but the docs read like dense legalese or half-finished bullet points. The result? Frustrated developers, stalled projects, and endless "Do you know how this works?" Slack threads.

Over the past two years, I've owned the documentation for platforms like Shiftconnector, Augmentir, and GoCanvas—everything from onboarding guides and wikis to SOPs for critical workflows. I've hosted admin and key-user trainings to ensure teams get maximum ROI from these tools. In this post, I'll share the principles and practices that turned our docs from a clogged knowledge pipeline into an engine for speed and confidence.

1. Start with Your Audience: Personas, Not Features

Who's Reading?
  • New Admins need a gentle ramp-up: account setup, permissions, high-level architecture.
  • Key Users (Ops, Maintenance, Quality) want task-specific quick wins: "How do I trigger an AR overlay?" or "Where do I find audit logs?"
  • Developers & Integrators crave API examples, payload schemas, and sample scripts.
Actionable Tip: Kick off each guide with a one-sentence "This doc is for…" line. It cuts confusion and sets expectations up front.

2. Organize by Task, Not by Menu

The Pitfall of Menu-Driven Docs

Listing every dropdown and checkbox in the UI might feel thorough, but it's overwhelming. Users don't care about menus—they care about outcomes.

Task-Centered Structure
  • Onboarding Guide:
    1. Create your first admin user
    2. Configure user roles
    3. Connect your first data source
  • Deployment Doc:
    1. Install prerequisites (VM, Docker, dependencies)
    2. Run the setup script
    3. Verify connectivity and run smoke tests
  • SOP Walk-Through (Shiftconnector Bypass Interlocks):
    1. Navigate to the Workflow module
    2. Select the "Bypass Interlocks" process
    3. Complete each safety confirmation step
    4. Review the generated audit log for compliance

By anchoring each section around a real goal, your reader dives straight into action without hunting for the right fragment.

3. Real-Life Example: Bypass Interlocks at an Ethanol Plant

At one ethanol facility I support, safety interlocks require manual verification during certain maintenance windows. I developed an SOP document that walks operators through:

  1. Launching the Workflow: A step-by-step screenshot tour of the "Bypass Interlocks" workflow in Shiftconnector.
  2. Performing Safety Checks: Clear instructions (with callouts) on verifying valve positions and recording sensor readings.
  3. Digital Sign-Off: How to capture operator ID and timestamp each approval.
  4. Audit Review: Locating the final audit log entry and exporting it for regulatory reporting.

Because the guide is task-focused and richly annotated, new shift leads can complete the bypass safely in under 5 minutes—down from 20 minutes of ad-hoc email exchanges and paper logs.

4. Use Visuals and Code Side-by-Side

Why It Works
  • Screenshots with Callouts bridge the gap between description and click-practice.
  • Code snippets paired with sample API requests let developers copy-paste and experiment immediately.

Example for Developers

// Trigger "Bypass Interlocks" workflow via API
POST /api/workflows/start
{
    "workflowId": "bypass_interlocks_etoh",
    "operatorId": "user_123",
    "parameters": {
        "plant": "EthanolPlantA",
        "step": "safety_check_complete"
    }
}

Next to it: a screenshot highlighting the "Start Workflow" button and the JSON payload fields in Shiftconnector's UI. Instant clarity, zero guesswork.

5. Create Living Docs with Version Control

The Challenge

Static PDFs or wiki pages age quickly—especially in fast-moving environments where new features roll out weekly.

The Solution
  • Doc-as-Code: Maintain markdown files in Git—same PR/review process as your code.
  • Changelog Front and Center: At the top of each guide, include "What's Changed" so returning readers know what's new.
  • Automated Publishing: Hook your docs repo into a static site generator (like Docusaurus) for instant updates.

6. Measure and Iterate

Metrics That Matter
  • Time to First Success: How long does it take a new admin to complete initial setup?
  • Support Ticket Volume: Which doc pages correlate with spikes in help requests?
  • Wiki Page Views & Search Queries: Identify dead-ends or highly trafficked topics.
Feedback Loops

Embed quick feedback widgets or run short post-training surveys to uncover friction points. Then iterate documentation in your next sprint.

Conclusion

Great documentation isn't an afterthought—it's the superhighway connecting your product to the people who use it. By writing for specific personas, organizing around tasks, surfacing real-world examples like bypass interlocks in Shiftconnector, using visuals and living docs, and measuring impact, you transform confusion into clarity.

Whether you're rolling out a new workflow or building an API integration, invest in docs developers actually use—and watch your team accelerate beyond what you thought possible.
